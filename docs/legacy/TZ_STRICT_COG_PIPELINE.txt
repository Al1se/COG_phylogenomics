СТРОГОЕ ТЗ: пайплайн COG0086 (курсовой проект)
Дата: 2026-03-24

0) Назначение
- Получить воспроизводимый набор файлов для анализа консервативности COG0086 (RpoC, β′):
  1) большое MSA с добавленными структурами,
  2) фильтрация только структурного мусора (COG-последовательности не удалять),
  3) дерево в Newick,
  4) MSA, переставленное по порядку листьев дерева,
  5) итоговые FASTA с массовым переименованием заголовков.

1) Жесткие требования
- COG-последовательности short-list (275 list) сохранять ВСЕ, без удаления.
- Фильтрация применяется только к структурным (xray/PDB-derived) последовательностям.
- MUSCLE запускать с флагом -output.
- Newick должен быть записан в отдельный файл .nwk.
- Перестановка MSA должна точно соответствовать порядку листьев Newick.

2) Входные файлы
- 275_COG0086.fasta
- newZN0086.fasta
- Скрипты:
  - 04_prepare_structure_fasta.py
  - 06_filter_msa_keep_all_cogs.py
  - 07_build_upgma_tree.py
  - 02_sort_fasta_by_newick.py
  - 08_rename_fasta_headers.py

3) Метод дерева (фиксируется явно)
- Метод: UPGMA (average linkage) по Hamming-distance на aligned FASTA.
- Инструмент: локальный скрипт 07_build_upgma_tree.py (scipy).
- Это рабочее техническое дерево для упорядочивания MSA и обзора консервативности.

4) Обязательные шаги пайплайна
Шаг A. Подготовить структуры
- Очистка структурного FASTA:
  - убрать gap-символы,
  - удалить короткие последовательности (по длине),
  - сформировать cleaned FASTA и отчет.

Шаг B. Собрать общий вход
- Объединить COG0086 short-list FASTA + cleaned structures FASTA.

Шаг C. Построить большое MSA
- MUSCLE super5:
  muscle -super5 <input> -output <msa>

Шаг D. Фильтрация (только структуры)
- Принудительно оставить все COG-id.
- Оставить структуры по score/threshold до целевого размера ~500 seq.

Шаг E. Построить дерево
- Из filtered MSA построить UPGMA Newick.

Шаг F. Переставить MSA по дереву
- Отсортировать FASTA по листьям дерева.

Шаг G. Переименовать заголовки
- Формат: ОБЩИЙID@phylogeny (или по mapping TSV).

5) Критерии приемки (Definition of Done)
- В итоговой папке присутствуют файлы:
  - COG0086.275.raw.fasta
  - COG0086.structures.cleaned.fasta
  - COG0086.mega.input.fasta
  - COG0086.mega.msa.fasta
  - COG0086.mega.msa.filtered_keepcogs.fasta
  - COG0086.mega.tree.upgma.nwk
  - COG0086.mega.msa.filtered.reordered.fasta
  - COG0086.mega.msa.filtered.reordered.renamed@phylogeny.fasta
  - RUN_SUMMARY.txt
- Проверка количества записей:
  - число COG в filtered_keepcogs >= числу COG во входном short-list,
  - итоговое число последовательностей близко к target_total.
- 02_sort_fasta_by_newick.py должен сообщать Matched records == Input FASTA records.

6) Нефункциональные требования
- Все команды и параметры сохранять в текстовых логах/README в папке прогона.
- Исходные входы не перезаписывать.
- Все шаги должны быть воспроизводимы вручную с CLI.

7) Важное по мотивам
- Искусственный motif=XXAAXX НЕ использовать как научный мотив.
- Реальные мотивы (например, NADFDGD, Zn-связывающие) выбирать вручную по общему выравниванию.
- motif-скрипт запускать только после ручной фиксации списка мотивов.
