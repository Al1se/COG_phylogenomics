COG0085/COG0086 курсовой проект — рабочие заметки
Дата: 2026-03-24

1) Контекст проекта
- Объект: бактериальные субъединицы РНК-полимеразы:
  - COG0085 = RpoB (β)
  - COG0086 = RpoC (β′)
- Текущий фокус: эволюционный/филогеномный анализ и консервативность функциональных участков.
- Важные функциональные регионы:
  - COG0086 (β′): Zn-связывающий участок (ZBD), каталитический мотив NADFDGD, Mg2+-связанный центр.
  - COG0085 (β): участок связывания рифамицинов + вариабельные соседние области.
- Есть риск загрязнения выборки: в MSA могут попадать другие субъединицы РНК-полимеразы.

2) Что уже есть в папке
- Скрипты:
  - cog-extractor-full.py (извлечение последовательностей из COG-таблицы и общей FASTA-базы)
  - getpdb.py (получение последовательностей по PDB)
- Данные:
  - COG0085.fasta, COG0086.fasta (+ 275_COG0085/275_COG0086)
  - ZN0085, ZN0086 (списки PDB)
  - ZN0085.fasta, ZN0086.fasta (полученные PDB-последовательности)
  - cog-24.cog.csv, 2296Genomes.prot.fasta и др.

3) Цель пайплайна (минимальный, аккуратный, мало кода)
Шаг A. Подготовка входов
- Выгрузить последовательности COG0085/COG0086.
- Добавить структурно охарактеризованные последовательности (из PDB).

Шаг B. MSA
- Выполнить MSA через MUSCLE (локально).
- Подтвержденный формат запуска MUSCLE:
  muscle -align ./ZN0085.fasta -output ./test_pdb.fasta
- Отдельно для COG0085 и COG0086.

Шаг C. Очистка от нерелевантных последовательностей
- Нужен скрипт фильтрации по похожести на профиль/референс в выравнивании.
- Идея: оставить только последовательности, у которых доля совпадений в информативных колонках >= порога.
- Порог сделать параметром CLI.

Шаг D. Дерево
- Построить дерево (как раньше через ngphylogeny.fr или локально).
- Экспорт в Newick.

Шаг E. Пересортировка MSA по дереву
- Нужен скрипт, который упорядочивает последовательности в FASTA по порядку листьев Newick.

Шаг F. Мотивы и локальные окна
- Нужен скрипт для вырезки окна вокруг мотива формата:
  - --XXAAXX-- (идея: motif + flanks)
- Параметры: motif, длина фланков слева/справа, режим строгий/с допусками.

Шаг G. Визуализация и интерпретация
- Финальная MSA + дерево + annotation мотивов/сайтов Zn/Mg.
- Анализ консервативных/вариабельных позиций и lineage-specific insertions.

4) Минимальный набор новых скриптов (реализовано)
- 01_filter_msa_by_similarity.py
  Вход: aligned.fasta, reference_ids или consensus, threshold
  Выход: filtered.fasta (+ short report)

- 02_sort_fasta_by_newick.py
  Вход: aligned_or_raw.fasta, tree.nwk
  Выход: reordered.fasta

- 03_extract_motif_windows.py
  Вход: fasta, motif, flank_left, flank_right
  Выход: motif_windows.fasta (только окна вида XX[motif]XX)

Принцип: все 3 скрипта максимально короткие, аргументы через argparse, без тяжелых зависимостей (по возможности стандартная библиотека + Biopython при необходимости).

5) Обновленные существующие скрипты (реализовано)
- cog-extractor-full.py
  CLI-скрипт для извлечения всех последовательностей заданного COG.
- cog-extractor-short-list.py
  CLI-скрипт для извлечения COG-последовательностей только по short-list (275list).
- cog_extractor_common.py
  Общее ядро парсинга FASTA/COG и записи результатов.
- getpdb.py
  Загрузчик PDB_chain -> FASTA с повторами запросов и списком ошибок.
- pdb_equal.py
  Сравнение списков PDB_chain (несколько режимов set-diff).

7) На что смотреть в анализе
- Сохраняется ли NADFDGD в COG0086 после фильтрации.
- Сохраняются ли Zn-связывающие мотивы (ZBD / Zn-finger) в ожидаемых кластерах.
- Где расположены вариабельные вставки относительно дерева и таксонов.
- Какие позиции вокруг rifamycin-site в COG0085 наиболее изменчивы.
